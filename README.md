# Metis 設計レビュー資料

このリポジトリは、卒業制作 **Metis** の設計レビュー用資料です。

GitHubに慣れていない方でも確認しやすいように、通常のGitHub画面ではなく、**Webサイト形式のレビュー資料**として閲覧できるようにしています。

---

## まず見る場所

基本的には、以下のGitHub PagesのURLから閲覧してください。

```text
https://5b-projects.github.io/review_Repository/
```

上のURLを開くと、MkDocsで生成されたWebサイト形式の資料が表示されます。

> まだページが表示されない場合は、GitHub Actionsによる公開処理が完了していない可能性があります。数分待ってから再度開いてください。

---

## この資料の目的

元の開発リポジトリには、設計資料が多数あります。

- 画面遷移図
- 画面レイアウト
- コンポーネントツリー図
- 画面データ要求
- ER図
- Supabaseデータ配置
- DBスキーマ設計
- ADR

ただし、すべてを一度に確認すると量が多すぎるため、このリポジトリでは **先生レビューに必要な範囲へ絞って再構成**しています。

特に、今回は以下の設計図を中心にしています。

```text
05 コンポーネントツリー図
06 画面データ要求
07 ER図
09 Supabase配置
10 DBスキーマ設計
```

## 見ていただきたい順番

GitHub Pagesを開いたら、上部メニューから以下の順に確認してください。

### 1. 入口

この資料全体の目的と、レビュー対象範囲を説明しています。

### 2. 05 コンポーネントツリー図

画面をReactのUI部品として、どのように分割するかを示す図です。

最初に重点的に見ていただきたいのは以下です。

| 画面ID | 画面名 | 確認してほしいこと |
|---|---|---|
| SC002 | ホーム | 新規学習開始・学習再開への導線が自然か |
| SC006 | ガイド生成 | ゴール入力、タグ選択、生成中表示の構成が分かりやすいか |
| SC007 | ガイド進行 | 学習本文、AI相談、進捗、評価の配置が分かりやすいか |
| SC008 | 学習ガイド終了 | 完了結果、AI評価、教材化導線が自然か |
| SC009 | 教材詳細 | 教材閲覧、学習開始、レビュー、購入導線が過不足ないか |

### 3. 06 画面データ要求

各画面で必要になるデータを、以下の3つに分けて整理しています。

- 表示するデータ
- ユーザーが入力・操作するデータ
- 保存・更新するデータ

ここでは、画面に必要な情報が不足していないかを確認していただきたいです。

### 4. 07 ER図

画面データ要求をもとにした、データ構造の図です。

初回レビューでは、全体を細かく確認するより、以下の流れが自然につながっているかを見てください。

```text
利用者
  ↓
ゴール入力
  ↓
ガイド生成ジョブ
  ↓
学習ガイド
  ↓
ガイド進捗
  ↓
AI相談・評価
  ↓
ガイド完了
  ↓
教材化
  ↓
教材
```

### 5. 09 Supabase配置

どのデータをSupabaseのどのサービスに置くかを説明しています。

例として、以下のように分けています。

| データ | 配置先 |
|---|---|
| 通常のテーブルデータ | Postgres |
| ログイン情報 | Supabase Auth |
| 教材本文・画像・AI入出力 | Storage |
| 進捗通知 | Realtime |
| APIキーなどの秘密情報 | Vault |
| 非同期処理 | pgmq + jobs |

### 6. 10 DBスキーマ設計

実装時のテーブル名、カラム名、RLS、マイグレーション順序などを整理しています。

細かいSQL本文ではなく、設計方針として妥当かを確認していただきたいです。

## 初回レビューで特に確認してほしいこと

以下の観点で見ていただけると助かります。

1. コア機能の流れが自然か
2. 画面構成が分かりやすいか
3. 画面に必要なデータが不足していないか
4. 画面データ要求とER図が対応しているか
5. データ保存先の分け方が妥当か
6. 卒業制作として、実装範囲が広すぎないか

## 初回レビューでは細かく見なくてよい範囲

以下は重要ではありますが、最初のレビューでは補助扱いにしています。

| 範囲 | 理由 |
|---|---|
| 課金・決済 | コア体験ではなく、論点が大きいため |
| 管理者画面 | 運用機能であり、初回レビューでは範囲が広がりすぎるため |
| 監査ログ | 管理者機能とあわせて別紙確認でよいため |
| 組織管理 | 将来拡張要素が強いため |
| 通知・フォロー | 利便性・SNS寄り機能のため |
| 08 テーブルレイアウト | 現在はDBスキーマ設計へ吸収済みのため |

## 画像が表示されない場合

画像は、元のPrivateリポジトリ `5B-Projects/metis` からGitHub Actionsでコピーして表示する構成です。

画像が表示されない場合は、以下のどちらかが原因です。

1. GitHub Actionsの公開処理がまだ完了していない
2. `METIS_REPO_TOKEN` が設定されていない

この場合、学生側で設定を確認します。

## 学生側メモ

このリポジトリは MkDocs で構成しています。

- 設定ファイル: `mkdocs.yml`
- 公開処理: `.github/workflows/deploy-mkdocs.yml`
- 本文: `docs/*.md`
- 画像コピー元: `5B-Projects/metis`

GitHub Pages公開には、`review_Repository` 側で以下のSecretが必要です。

```text
METIS_REPO_TOKEN
```

Fine-grained personal access tokenを使う場合は、`5B-Projects/metis` に対する `Contents: Read-only` 権限で足ります。
