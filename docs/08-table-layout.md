# 08 テーブルレイアウト

## 初回レビューでの扱い

`docs/design/08-table-layout/` は、現在のレビュー用資料では **除外** 扱いにします。

理由は、元リポジトリ側で `08-table-layout/` が `10-database-schema/` への吸収元として扱われているためです。

## 先生へ見せる場合の注意

この資料をそのまま出すと、以下の混乱が起きやすいです。

- ER図の正本とテーブルレイアウトの旧記述が混ざる
- DB物理設計の正本がどれか分かりにくくなる
- マイグレーションとの整合性確認が必要になる

## 方針

レビュー用リポジトリでは、08を直接見せるのではなく、以下へ集約します。

| 代替資料 | 目的 |
|---|---|
| 07 ER図 | 論理テーブルと関係を確認する |
| 09 Supabase配置 | Supabaseのどのサービスに置くか確認する |
| 10 DBスキーマ設計 | 物理名規約、RLS、migration順序を確認する |

## 結論

08は「履歴として残す資料」であり、先生レビュー用の正本としては扱わない方が安全です。
