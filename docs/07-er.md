# 07 ER図

## 位置づけ

`docs/design/07-er/` は、画面データ要求書から導いた ER 図を管理する設計図です。

- 元リポジトリでは `all-er-diagrams.puml` が ER 図の正本です。
- `image/all-er-diagrams.png` はレンダリング結果です。
- 個別の分割 ER 図はレガシー化され、全画面統合 ER 図へ統合されています。

ただし、全画面統合ER図は初回レビューには重いため、このページでは先に **コア範囲だけのER図** を表示します。

---

## コア範囲ER図

初回レビューでは、以下の流れに必要なテーブルだけを見ます。

```text
利用者
  ↓
ゴール入力
  ↓
ガイド生成
  ↓
ガイド進行
  ↓
AI相談・評価
  ↓
ガイド完了
  ↓
教材化
  ↓
教材
```

![コア範囲ER図](../assets/er/core-er-diagram.svg)

!!! note "この図の目的"
    完全版ER図から、先生レビューで最初に確認したい範囲だけを抜粋した図です。  
    正本を置き換えるものではなく、全体ERを見る前の入口として使います。

---

## コア範囲のエンティティ

| 領域 | 主なテーブル | レビュー観点 |
|---|---|---|
| 利用者・前提設定 | users, user_profiles, learning_style_defaults, technology_tags | ガイド生成に必要な利用者属性・学習スタイル・タグが揃っているか |
| ガイド生成 | goal_drafts, jobs, generation_jobs, ai_executions, guardrail_results | AI生成の非同期処理、安全性検査、生成状態を追跡できるか |
| 学習ガイド・進捗 | guides, guide_chapters, guide_steps, guide_progress, chat_messages | ガイド本文と利用者ごとの進捗を分けて管理できるか |
| AI評価・教材化 | evaluation_jobs, evaluation_results, guide_completions, ai_evaluation_summaries, material_creation_jobs | 評価結果と教材化の根拠を保存できるか |
| 教材 | materials, material_versions, material_sections, material_chapters, material_tags | 完了ガイドから教材として公開・閲覧できる構造か |

---

## コアから外した範囲

以下は完全版ER図には含まれますが、初回レビューでは別紙扱いにします。

| 領域 | 理由 |
|---|---|
| 課金・カート・決済・利用権限 | 教材購入には必要だが、ガイド生成・進行の中核から外れるため |
| 管理者・監査ログ | 運用設計として重要だが、初回レビューでは論点が広がりすぎるため |
| 組織・組織API認証情報 | 将来拡張要素が強いため |
| 通知・フォロー・お気に入り | 利便性・SNS寄り機能のため |
| 検索ログ・検索結果スナップショット | 分析・改善寄りのデータのため |
| 規約同意・外部アカウント連携 | 認証補助として重要だが、コア体験の説明からは外れるため |

## レビュー観点

1. コア機能のデータが、画面データ要求と対応しているか。
2. AI生成・AI評価の入力/出力本文をDBへ直書きしない方針でよいか。
3. ジョブ共通親 `jobs` とドメイン別詳細テーブルの分割が妥当か。
4. ガイド進捗を `guide_progress`、`chapter_progress`、`step_progress` に分ける粒度が妥当か。
5. 教材化ジョブと教材本体の境界が妥当か。
6. 課金・管理者・監査ログを初回レビューから外す判断で問題ないか。
