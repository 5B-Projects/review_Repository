# レビュー用構成

## 推奨するレビュー範囲

初回レビューでは、全設計を一度に見せるのではなく、以下の順に確認します。

![初回レビューの確認順](assets/review/core-review-flow.svg)

| 順番 | 資料 | 見ること |
|---|---|---|
| 1 | 05 コンポーネントツリー図 | 画面をどの粒度で部品化するか |
| 2 | 06 画面データ要求 | 画面ごとの表示・入力・保存・更新契機 |
| 3 | 07 ER図 | 画面データ要求を支えるテーブルと関係 |
| 4 | 09 Supabase配置 | DB、Storage、Auth、Edge Functionsの分担 |
| 5 | 10 DBスキーマ設計 | 実装時のテーブル、制約、RLS方針 |

## 画像ファイルの扱い

このレビュー資料で使う画像は、`review_Repository` の `docs/assets/` 配下に直接含めます。

| 配置先 | 用途 |
|---|---|
| `docs/assets/review/` | レビュー順序や全体方針の図 |
| `docs/assets/component-tree/` | コンポーネントツリー図 |
| `docs/assets/er/` | ER図・ER概要図 |

この方針により、GitHub Actions は `review_Repository` だけをチェックアウトしてMkDocsをビルドできます。  
`5B-Projects/metis` から画像をコピーするための `METIS_REPO_TOKEN` は不要です。

## 先生へ伝える説明文案

> 本リポジトリは、卒業制作 Metis の設計レビュー用に、元リポジトリから必要資料を抽出・再構成したものです。  
> 元リポジトリには全画面・全ER・全ADRを含む詳細設計がありますが、初回レビューでは量が多いため、特に `05-component-tree` 以降の設計図を中心に整理しました。  
> まずは、コア機能である「ゴール入力 → ガイド生成 → ガイド進行 → AI相談・評価 → 教材化 → 教材閲覧」について、画面分割・データ要求・ER・DB方針がつながっているかをご確認いただきたいです。
