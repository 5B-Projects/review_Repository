# 06 画面データ要求

## 位置づけ

画面データ要求は、画面ごとに以下を整理する設計図です。

- 表示データ
- 入力データ
- 保存・更新データ
- 保存・更新契機
- 備考・ADRとの関係

このページでは、初回レビュー対象として SC006〜SC009 を中心に抜粋します。  
特に、次ページのER図へつなげるため、**画面で起きる操作がどのデータ更新につながるか** を見ます。

---

## コア画面のデータ連鎖

| 流れ | 画面 | 入力・操作 | 保存・更新される主なデータ |
|---|---|---|---|
| 1 | SC006 ガイド生成 | ゴール入力、タグ選択、学習スタイル設定 | ゴール下書き、選択タグ、学習スタイルスナップショット |
| 2 | SC006 ガイド生成 | 構成案確認、生成開始 | 生成ジョブ、構成案、AI実行履歴、安全性検査結果 |
| 3 | SC007 ガイド進行 | 章/ステップ閲覧、完了チェック | ガイド進捗、章進捗、ステップ進捗、学習履歴 |
| 4 | SC007 ガイド進行 | AI相談、コード評価 | チャット履歴、AI実行履歴、評価ジョブ、評価結果 |
| 5 | SC008 学習ガイド終了 | 完了確認、教材化操作 | ガイド完了記録、AI評価サマリー、教材化ジョブ |
| 6 | SC009 教材詳細 | 教材閲覧、学習開始、レビュー、作者操作 | 教材学習履歴、レビュー、公開状態、作者メモ、購入/閲覧権限 |

---

## SC006 ガイド生成

### 表示

ステッパーの現在位置、ゴール入力例、ゴール入力の必須表示、入力エラー文言、テンプレートゴール、学習スタイル初期値、タグ候補、タグカテゴリ、推奨タグ、タグ説明、前提知識、タグ選択状態、選択タグ数、選択上限、構成案、生成ステータス、生成進捗率、推定待ち時間、生成サマリー、ブロック理由、環境ファイル生成状態、生成失敗状態。

### 入力

ゴール本文、テンプレート選択、タグ検索、選択タグ、推奨タグの全選択・全解除操作、生成時学習スタイル上書き、詳細設定ファイル、カスタムAPI利用有無、下書き保存操作、生成開始操作、構成案の承認・修正・再生成操作、生成キャンセル操作、生成完了後の保存して学習開始操作。

### 保存・更新

ゴール下書き、安全性チェック結果、選択タグ、生成ジョブ、学習スタイルスナップショット、構成案、ガイド本文、環境構築ファイルジョブ、ガードレール結果、生成版。

### イベントと保存先

| イベント | 保存・更新 | 主な保存先候補 |
|---|---|---|
| ゴール本文を入力する | 下書き本文、入力状態 | `goal_drafts` |
| タグを選択する | 選択タグ、タグ数、推奨タグ反映状態 | `goal_draft_tags`, `technology_tags` |
| 学習スタイルを上書きする | 生成時点のスタイルスナップショット | `generation_jobs` |
| 構成案を承認する | 構成案の版、承認状態 | `guide_outline_versions` |
| 生成を開始する | ジョブ、進捗、失敗理由 | `jobs`, `generation_jobs` |
| AI生成が完了する | 生成済みガイド、章、ステップ | `guides`, `guide_chapters`, `guide_steps` |
| 安全性検査で止める | ブロック理由、検査結果 | `guardrail_results` |

### レビュー観点

- ゴール入力からタグ選択、構成案確認、生成完了までの状態が追えるか。
- 生成中・失敗・再生成の状態を保存できるか。
- 生成時の学習スタイル上書きが既定値と混ざらないか。

## SC007 ガイド進行

### 表示

ガイド概要、章一覧、現在章バッジ、進行状況、章、ステップ、学習目的、完了条件、完了チェック状態、次章ボタン活性状態、全体進捗、記事タイトル・本文、コードブロック、本文内の解説対象用語、環境作成状態、AIチャット履歴、AIドロワーの開閉状態、コード評価結果、未完了警告。

### 入力

ステップ完了チェック、次章へ進む操作、AIへの質問、選択テキスト・用語クリックによるAI解説依頼、コード評価開始操作、フィードバック利用操作、コードブロックのコピー操作、AI回答コピー操作。

### 保存・更新

ガイド進捗、章/ステップ完了状態、学習履歴、チャット履歴、AI評価ジョブ、実行評価、品質評価、フィードバック、ユーザーレベル推定、教材化候補。

### イベントと保存先

| イベント | 保存・更新 | 主な保存先候補 |
|---|---|---|
| ガイドを開く | 最終閲覧日時、現在章/ステップ | `guide_progress`, `learning_events` |
| ステップを完了する | ステップ完了状態、全体進捗率 | `step_progress`, `guide_progress` |
| 次章へ進む | 章完了状態、現在章 | `chapter_progress`, `guide_progress` |
| AIへ質問する | 質問、回答、文脈、AI実行ID | `chat_messages`, `ai_executions` |
| コード評価を開始する | 評価ジョブ、評価対象、状態 | `jobs`, `evaluation_jobs` |
| 評価結果を受け取る | スコア、評価コメント、改善提案 | `evaluation_results` |
| フィードバックを利用する | 利用ログ、反映先 | `feedback_usage_logs` |

### レビュー観点

- 章進捗とステップ進捗を分ける必要があるか。
- AI相談メッセージをガイド進捗に紐づける設計で十分か。
- コード評価結果とユーザーレベル推定の分離が妥当か。

## SC008 学習ガイド終了

### 表示

完了したガイド、技術タグ、ガイド完了バッジ、完了日時、進捗率100%表示、学習時間、総合評価、AI評価サマリー、次に伸ばすポイント、完了サマリー画像、パンくず表示、教材作成導線、復習導線、ガイド一覧導線。

### 入力

教材作成操作、復習操作、ガイド一覧へ戻る操作、お気に入り登録、レビューリクエスト、タグ付与確認。

### 保存・更新

ガイド完了状態、教材下書き、教材タイトル/説明/タグ候補、完成イメージ画像ジョブ、公開不可理由、レビュー依頼状態。

### イベントと保存先

| イベント | 保存・更新 | 主な保存先候補 |
|---|---|---|
| ガイド完了画面を表示する | 完了記録、完了日時、学習時間 | `guide_completions` |
| AI評価を表示する | 総合評価、評価サマリー、改善提案 | `ai_evaluation_summaries`, `evaluation_results` |
| 教材化を開始する | 教材化ジョブ、作成状態 | `jobs`, `material_creation_jobs` |
| 教材下書きを作る | タイトル、説明、タグ候補 | `materials`, `material_versions`, `material_tags` |
| 復習へ戻る | 現在章/ステップの復元 | `guide_progress` |

### レビュー観点

- ガイド完了と教材化の境界が明確か。
- 教材化は完了済みガイドに限定する方針でよいか。
- AI評価サマリー、評価項目、改善提案をどこまで保存すべきか。

## SC009 教材詳細

### 表示

パンくず、教材タイトル、説明、価格、評価値と星の視覚表示、レビュー数、受講者数、作者、作者マイページへの導線、最終更新日、言語、タグ、ゴール、学習内容の項目リスト、目次、公開範囲、完成イメージ画像、購入/閲覧権限、レビュー一覧、レビュー集計、AIコメント要約、作成者メニュー、作者メモ、エクスポート状態、コード参照可否、技術バージョン・作者OS。

### 入力

学習開始、カート追加、エクスポート形式選択、リンクコピー、レビュー投稿、公開/非公開切替、作者メモ、通報操作。

### 保存・更新

カート明細、教材学習履歴、レビュー投稿、レビュー集計再計算、公開/非公開状態、作者メモ、エクスポートジョブ、通報フォーム起動記録。

### イベントと保存先

| イベント | 保存・更新 | 主な保存先候補 |
|---|---|---|
| 教材詳細を開く | 閲覧履歴、権限確認 | `material_view_events`, `entitlements` |
| 学習開始する | 教材学習履歴 | `material_learning_histories` |
| カートに追加する | カート明細 | `cart_items` |
| レビューを投稿する | レビュー本文、評価値、集計更新 | `material_reviews`, review aggregate |
| 作者メモを保存する | 作者メモ | `material_author_notes` |
| 公開状態を切り替える | 公開/非公開状態、変更履歴 | `materials`, `material_status_histories` |
| 通報する | 通報下書き、問い合わせ導線 | `reports`, `inquiries` |

### レビュー観点

- 教材詳細に課金・レビュー・作者操作が集まりすぎていないか。
- 通常ユーザーの編集範囲を作者メモと公開状態に限定する方針が妥当か。
- 教材版、教材セクション、教材チャプターの分割が画面表示と一致しているか。

## ER図へ渡す確認ポイント

| 確認ポイント | ER図側で見る場所 |
|---|---|
| 画面表示に必要な値が正本として保存されるか | コア範囲ER図の各エンティティ |
| 非同期処理の状態が追えるか | `jobs` と用途別ジョブ詳細 |
| AI入出力の長文をDBへ直書きしない方針でよいか | `ai_executions` とStorage配置方針 |
| ガイド本文と利用者進捗が混ざっていないか | `guides` 系と `guide_progress` 系の分離 |
| 教材化の根拠が残るか | `guide_completions`, `ai_evaluation_summaries`, `material_creation_jobs` |
